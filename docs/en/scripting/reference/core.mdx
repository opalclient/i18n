# Core Proxies

These proxies handle general client interaction, notifications, and UI.

## Overlay Proxy
**Global Binding:** `overlay`

Allows scripts to register and manage custom "Dynamic Island" HUD elements.

<Method
    name="createIsland"
    signature="String createIsland(Object config)"
    description="Creates a new island and returns its unique ID. Config must include width, height, priority, and a render callback."
    params={[
        { name: "config", type: "Object", desc: "{ width, height, priority, render(posX, posY, width, height, progress) }" }
    ]}
    returns={{ type: "String", desc: "Unique Island ID." }}
/>

```javascript
const island = overlay.createIsland({
    width: 160,
    height: 28,
    priority: 10,
    render: (posX, posY, width, height, progress) => {
        renderer.roundedRect(posX, posY, width, height, 4, 0xFF1a1a1a);
        renderer.text("productsans-bold", "Custom HUD", posX + 10, posY + 14, 8, 0xFFFFFFFF);
    }
});
overlay.showIsland(island);
```

## mc — MinecraftProxy
**Global Binding:** `mc`

Null-safe access to Minecraft instances.

<Callout type="warning">
    Do not call methods on `mc.player` or `mc.world` directly — Fabric's intermediary mappings rename methods at runtime. Use the dedicated `player`, `world`, and `inventory` proxy globals instead. `mc` is for null checks and `interactionManager` access only.
</Callout>

| Property | Type | Description |
|---|---|---|
| `mc.player` | `ClientPlayerEntity \| null` | Null-check only. Use `player` global. |
| `mc.world` | `ClientWorld \| null` | Null-check only. Use `world` global. |
| `mc.interactionManager` | `InteractionManagerProxy` | Block and entity interaction (see Interaction reference). |

```javascript
if (mc.player === null || mc.world === null) return;
```
